# 股票庫藏追蹤系統（每日自動更新）

## 🎯 需求
建立一個網頁：

- 每天自動更新（無需手動）
- 使用 Python 計算股票庫藏績效
- 支援台股 / 美股
- 顯示：
  - 損益（$ / %）

- 部署成可公開存取的靜態網頁

### `transactions.csv` 目前限制

目前 `transactions.csv` 只支援「買進交易」，沒有賣出欄位。

目前必要欄位是：

```text
ticker,buy_date,buy_price,quantity,fee,tax,account,note
```

目前程式假設：

- 每一列都是買進
- `quantity` 是持有部位增加
- `transaction_cost = buy_price * quantity + fee + tax`
- 沒有 `sell_date`、`sell_price`、`side`、`realized_pnl` 這類欄位

所以目前不適合直接記錄：

- 部分賣出
- 全部賣出
- 當沖 / 來回交易
- 已實現損益拆分

如果之後要支援賣出，建議改成交易流水格式。

### 建議新版 `transactions.csv` 格式（buy / sell）

建議欄位：

```text
trade_date,ticker,side,price,quantity,fee,tax,account,currency,note
```

欄位說明：

- `trade_date`
  - 交易日期
- `ticker`
  - 股票代號，例如 `2330.TW`、`NVDA`
- `side`
  - 交易方向，固定用 `buy` 或 `sell`
- `price`
  - 成交單價
- `quantity`
  - 成交股數 / 單位數
  - 一律填正數，不靠正負號表達買賣
- `fee`
  - 手續費
- `tax`
  - 交易稅
  - 台股賣出通常有稅，美股可填 `0`
- `account`
  - 券商 / 帳戶名稱
- `currency`
  - 幣別，例如 `TWD`、`USD`
- `note`
  - 備註

設計原則：

- `buy / sell` 用 `side` 表示，不用讓 `quantity` 帶正負號
- `quantity` 永遠填正數，避免後續匯入或人工編修時出錯
- `fee`、`tax` 都獨立欄位保留，後面才好算真實成本與已實現損益
- `currency` 放在交易表裡，未來做多幣別彙總時比較乾淨
- 不在 CSV 直接存 `realized_pnl`
  - 已實現損益應由程式依照成本法計算
  - 避免人工回填數字和系統計算打架

建議範例：

```csv
trade_date,ticker,side,price,quantity,fee,tax,account,currency,note
2024-11-19,2330.TW,buy,1084.27,3000,0,0,broker1,TWD,first buy
2025-02-10,2330.TW,sell,1125.00,1000,0,3375,broker1,TWD,partial take profit
2026-01-28,0050.TW,buy,73.91,7000,0,0,broker1,TWD,core ETF
2026-03-18,NVDA,sell,126.80,5,1,0,broker2,USD,trim position
```

### 為什麼建議這種格式

這種格式的好處：

- 可以同時支援
  - 加碼
  - 減碼
  - 全部出清
  - 多帳戶
  - 台股 / 美股
- 後面可用同一套資料推導
  - 目前持股數
  - 平均成本
  - 已實現損益
  - 未實現損益
  - 現金流
- 如果未來要加
  - FIFO
  - 平均成本法
  - 已實現 / 未實現拆分
  - 匯率換算
  也比較好延伸

### 實作時建議

後續程式可先採這個最穩定版本：

- 持股成本法：`移動平均成本`
- 賣出時計算：
  - `realized_pnl = 賣出收入 - 對應成本 - fee - tax`
- 目前持股：
  - 由 `buy` 累加、`sell` 扣減

若之後你需要報稅或更精確還原每筆出場，再考慮升級成 `FIFO`
